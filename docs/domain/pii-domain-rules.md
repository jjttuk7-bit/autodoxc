# PII 도메인 룰 v0

> **상태**: 1차 초안 — 행정사 검토 필요.
> **적용 대상**: M4 PII 게이트(`02-data-assets §DA2`)의 3중 검사(정규식 + NER + LLM) 중 도메인 관점 보강.
> **목적**: false negative(누락) 우선 방지, false positive(과제거)도 균형 잡기.

## 분류

- **HARD-PII** — 개인정보보호법 정의 개인정보. 자동 마스킹 + 승격 차단
- **SOFT-PII** — 식별 보조 정보. 마스킹 권장 + 사용자 알림
- **CORP-ID** — 기업 식별정보. 사업자번호 등 + 마스킹 권장
- **CONTEXT** — 도메인상 PII가 아니지만 결합 시 식별 가능

## HARD-PII (자동 마스킹 강제)

| 종류 | 패턴 예시 | 정규식 1차 | 비고 |
|---|---|---|---|
| 주민등록번호 | 800101-1234567 | `\d{6}-\d{7}` | 7번째 자리 1·2·3·4 검증 추가 |
| 외국인등록번호 | 800101-5234567 | `\d{6}-\d{7}` | 7번째 자리 5·6·7·8 |
| 운전면허번호 | 11-12-345678-90 | `\d{2}-\d{2}-\d{6}-\d{2}` | |
| 여권번호 | M12345678 | `[A-Z]\d{8}` | 컨텍스트 의존 (학번 등 false positive) |
| 의료보험번호 | 12-3456789-12345 | `\d{2}-\d{7}-\d{5}` | |
| 카드번호 | 1234-5678-9012-3456 | `\d{4}-\d{4}-\d{4}-\d{4}` | Luhn 체크 |
| 계좌번호 | 110-123-456789 | 금융기관별 패턴 표 필요 | |
| 휴대전화 | 010-1234-5678 | `01[016789]-\d{3,4}-\d{4}` | |
| 이메일 (전체) | name@example.com | RFC 5322 변형 | 도메인만 보존 옵션 |

## SOFT-PII (마스킹 권장 + 사용자 확인)

| 종류 | 예시 | 비고 |
|---|---|---|
| 한국인 성명 (2~4자) | 김민수 | 단독으로는 식별성 낮음, 조합 시 ↑ |
| 상세 주소 | 서울 강남구 테헤란로 123 | 시·구 단위까지는 보존, 동·번지 마스킹 |
| 생년월일 | 1980. 1. 1. | YYYY-MM-DD 형식 |
| 학교명 + 학년 | 강남고등학교 3학년 | |
| 사업장 전화번호 | 02-1234-5678 | 도메인 컨텍스트 |
| IP 주소 | 192.0.2.1 | 회원 식별 가능 |

## CORP-ID (기업 식별)

| 종류 | 예시 | 정규식 |
|---|---|---|
| 사업자등록번호 | 123-45-67890 | `\d{3}-\d{2}-\d{5}` (체크섬 검증) |
| 법인등록번호 | 110111-1234567 | `\d{6}-\d{7}` (주민번호와 충돌 — 컨텍스트로) |
| 회사명 (전체) | (주)○○○○ | LLM 분류 |
| 본점 주소 | 서울 ○○구 ○○동 1-1 | |
| 대표자명 | (자연인 성명과 동일 처리) | |

## CONTEXT (결합 시 식별 가능)

- 직책 + 부서 + 회사 → 1명 식별 가능
- 정확한 처분일자 + 기관 → 사건 식별
- 채용 경력 (회사명 + 기간) → 이력서 식별

행정문서 본문에 자주 나오므로 **무차별 마스킹 금지**. 라이브러리 승격 시점에만 LLM 분류로 결합 위험 평가.

## 도메인 특화 false positive (마스킹 금지)

다음은 PII처럼 보이지만 행정문서에서 보존되어야 함:

| 패턴 | 보존 이유 |
|---|---|
| 법령 인용의 조·항·호 숫자 (`제22조 제3항`) | 법령 인용은 마스킹 금지 |
| 공공기관 코드 (`B0000000`) | 행정 표준 코드 |
| 통계 비율·금액 (`35%`, `1,500만원`) | 정량 효과 표현 |
| 표준 양식 번호 (`별표 4`, `표 3-1`) | 양식 식별 |
| 인터넷 URL (외부 출처) | 인용 출처 |
| 우편번호 | 행정 구역 식별 — 마스킹 불필요 |
| 차량 등록 일반 형식 (`서울12가3456`) | 위반·고지 문서에 필요 (개별 차량 식별이지만 행정 사유) |

## 마스킹 표기

- 자리수 보존: `김민수` → `김**` (2자리 표시)
- 자리수 마스크: `010-1234-5678` → `010-****-5678`
- 전체 익명화: `123-45-67890` → `[사업자번호]`
- 라이브러리 승격 시 골격에서는 **필드 메타데이터만 남기고 값 제거**: `{ field: "company_name", type: "text" }`

## 3중 검사 적용 순서

1. **정규식** — 위 HARD-PII 패턴 + 체크섬 검증
2. **NER** — 한국어 NER 모델로 인명·기관·지명 추출
3. **LLM** — 1·2가 못 잡은 컨텍스트 결합 PII 분류 (Sonnet)

**hit 정책**: 하나라도 HARD-PII 분류 시 마스킹 강제. SOFT/CORP는 점수 조합 (정규식 + NER + LLM 합산 0.7 이상 시 사용자 알림).

## 행정사 검토 요청

1. **누락된 HARD-PII** — 위에 빠진 신분증명서 종류 (군번, 학번 등)
2. **false positive 추가** — 마스킹되면 안 되는 행정 표기 추가
3. **마스킹 표기 표준** — 부처 관행과 일치하는지
4. **승격 정책** — 어느 수준에서 PII로 분류해야 하는지 도메인 의견
