# 골격 검증 룰셋 v0

> **상태**: 1차 초안 — 행정사 검토 필요.
> **적용**: #1b SkeletonComposer 출력 검증 + SelfReviewer 골격 정합성 항목.
> **분류 기준**: (1) 법령 강제(불가침) (2) 부처 관행(가능변경) (3) 행정사 관행(유연).

## 분류 범례

- **L** Law-strict — 법령이 명시한 필수 섹션. 누락 시 신고·허가 자체가 무효
- **A** Agency-conventional — 부처가 표준 양식으로 정한 섹션. 누락 시 보완 요구
- **P** Practice-pragmatic — 행정사 실무 관행. 유연

## 공통 검증 규칙

### CR-1 법령 섹션(**L**) 누락 시
- SelfReviewer가 **blocker** 등급으로 보고
- DraftWriter는 빈 stub이라도 섹션을 생성해야 함 (사용자가 채우게)

### CR-2 섹션 순서
- L 섹션은 명세된 순서를 강제
- A·P는 행정사 관행에 맞춰 재정렬 가능

### CR-3 섹션 제목 표준화
- 번호 + 마침표 + 공백 + 제목: `1. 고용 대상 업종`
- 영문 번호(`A.`, `i.`) 금지
- 하위 섹션은 `1.1`, `가.`, `(1)` 순서로

### CR-4 빈 섹션
- 모든 paragraph가 `empty` status면 SelfReviewer가 **warning** 발송
- 사용자가 명시적으로 "이 섹션 빈 채로 진행" 선택 시 OK

### CR-5 골격 학습 후 검증
- #8 SkeletonLearner가 라이브러리에 등록 전 위 룰 자동 적용
- L 누락 골격은 승격 차단
