# 사용자 페르소나 v0

> **상태**: 1차 초안 — 사용자(시니어 엔지니어 PM)·행정사 검토 필요.
> **사용**: UX 의사결정 기준, 시나리오 정의, M5 도메인 검증의 사용자 관점.

## 페르소나 1 — 1인 행정사 김민호 (45세)

### 배경
- 경력 12년, 2017년 개업
- 사무소: 본인 + 사무직원 1명 (배우자)
- 주력: 외국인 고용 / 영업신고 / 행정심판
- 일평균 문서 작성: 3~5건

### 도구·기술 환경
- 한글(HWP) 위주, MS Word 일부
- Windows 11, 듀얼 모니터
- 정부24 / 민원24 / 국가법령정보센터 자주 사용
- 새 도구 학습 의지 보통 — "확실히 시간 절약되면 도입"

### 통점
- 같은 문서를 매번 처음부터 — 양식 재활용은 본인 컴퓨터 파일 검색
- 의뢰인이 자료를 제때 안 주거나 부정확하게 줌
- 법령 개정 후 양식 변경 추적이 부담
- 의뢰인별 PII 관리가 수작업

### 워크벤치에서 기대하는 것
- "이 의뢰인의 사실관계만 말하면 80% 채워지는 초안"
- "내가 자주 쓰는 표현·골격을 자동 학습"
- "법령 인용은 자동 + 정확"
- "한 화면에서 사실관계 / 본문 / 근거 다 보임"

### 부담스러워하는 것
- 회원가입 + 신용카드 입력 → 무료 체험 5건 정도 후 결제 안내가 자연스러움
- 클라우드에 의뢰인 PII 업로드 → 자체 DB / 국내 호스팅 / PII 자동 마스킹 강조 필요
- AI가 만든 문서를 그대로 내는 것 → 항상 검토 가능한 흔적 (annotation status)

## 페르소나 2 — 중소 법무팀 사원 박지영 (32세)

### 배경
- 직원 80명 제조업체 법무 담당 1.5명 중 1명
- 법학과 출신, 사법시험 준비 경험
- 외주 변호사·행정사 의뢰 관리
- 사내 문서: 통지서·내용증명·이의신청·인허가 신청서

### 도구·기술 환경
- MS Word, 사내 PDF 결재 시스템, Slack
- 외주 행정사와 메일·전화로 협업
- Google Workspace 익숙, Notion 사용

### 통점
- 외주 비용 대비 단순 통지·이의신청은 사내에서 처리하고 싶음
- 부서별 사실관계 수집이 비효율 (질문→답변→재질문)
- 매번 외주 변호사가 다른 톤·골격 → 일관성 부족

### 워크벤치에서 기대하는 것
- "사내 표준 톤·골격으로 자동 작성"
- "사실관계 누락 시 어떤 질문을 부서에 던질지 시스템이 알려줌"
- "외주 비용 절감 KPI에 기여"

### 부담스러워하는 것
- 사내 PII 외부 전송 → 자체 호스팅 옵션 또는 PII 마스킹 강력
- 너무 자유도 높은 도구 → 부서 표준화 어려움 → 사무소 골격 라이브러리 강제 적용 기능 필요

## 페르소나 3 — 행정사 사무소 대표 이수진 (52세)

### 배경
- 본인 + 행정사 2명 + 사무직원 2명 사무소 운영
- 주력: 외국인·법인 분야
- 신규 직원 교육 부담 高

### 워크벤치에서 기대하는 것
- "사무소 표준 골격 강제 적용" (개인 라이브러리보다 사무소 라이브러리 우선)
- "신입 직원 작성 → 시니어 검토 워크플로우"
- "월별 작성 통계·외부 비용 절감 효과 가시화"

### 부담스러워하는 것
- 직원별 권한 관리 복잡 — 사무소 단위로 일괄 적용 필요
- 외주 변호사·세무사와 PII 공유 시 책임 소재

## 공통 인사이트 — UX 설계 영향

| 인사이트 | 반영 위치 |
|---|---|
| 한글(HWP) 출처가 다수 | DA4 HWP 파서(B0-5)와 HWPX 변환 우선 |
| 본인 컴퓨터의 양식 재활용이 워크플로우의 시작 | 시작 화면 자유 입력 + 드래그앤드롭 (`03-ui-model §1`) |
| PII 신뢰 우려 | PII 게이트 강조, 자체 DB 사용 강조, 외부 전송 명시 |
| 사무소 단위 운영 | DA2 personal/shared 2계층 (ADR 0008) |
| 일관성 — 사무소 표준 강제 | 라이브러리 탭에서 "이 골격 강제 적용" UX (`03-ui-model §4.4`) |
| AI 산출물 검토 가능성 | 5가지 status 색상 + annotation refs (`03-ui-model §2.1`) |
| 사실관계 수집 비효율 | Conversational discovery (`02-data-assets §A-3` 원칙) |
| 신규 직원 교육 | 시드 골격 라이브러리 + 풀텍스트 검색 (DA3) |
