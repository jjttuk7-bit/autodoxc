# 행정문서 톤 가이드 v0

> **상태**: 1차 초안 — 행정사 검토 필요.
> **적용 대상**: DraftWriter(#6)의 본문 생성 + 시스템 UI 문구.
> **원칙**: "공손한 어조" 같은 추상이 아닌 **표준 예시·금지 예시**로 직접 기술.

## 1. 어체

### 1.1 본문 어체

- **객관체 + 경어체 혼합**: 신고/허가서·계획서는 객관체("…에 해당한다", "…를 제출한다"), 통지서·청구서·진정서는 경어체("…해 주시기 바랍니다")
- **현재형 우선**: "…하였습니다" 대신 "…합니다", 단 과거 사실 진술은 과거형 유지

### 1.2 어미 표준

| 문서 유형 | 끝맺음 어미 표준 |
|---|---|
| 계획서·신청서 본문 | "…한다" / "…이다" / "…된다" |
| 청구서·통지서 | "…하시기 바랍니다" / "…해 주십시오" |
| 의견서·진술서 | "…합니다" / "…이라고 사료됩니다" |
| 신고서 본문 | "…한다" / "…에 해당한다" |
| 계획서 결미 | "…할 계획이다" / "…할 예정이다" |

### 1.3 금지 표현

- ❌ 1인칭: "저희가", "우리가", "내가" → ⭕ "본 사", "신청인", "청구인"
- ❌ 구어체: "있구요", "그래서요", "~하구요" → ⭕ "있으며", "따라서", "…하고"
- ❌ 감정어: "안타깝게도", "매우 슬프게", "정말 화가 나는" → 제거
- ❌ 비속어·강조 강세: "절대로", "무조건", "당연히" → 객관 표현으로
- ❌ 영어 직역체: "~에 대해서" 남용 → "~에 관하여" / 생략
- ❌ "필자" / "기자" / "당사" 같은 다른 도메인 표기

### 1.4 인칭·자기 지칭

| 발신 주체 | 자기 지칭 |
|---|---|
| 회사/법인 | "본 사" / "당사" (계약 문서는 "갑/을") |
| 개인 사업자 | "본인" / "신청인" |
| 청구인 | "청구인" / "청구인 본인" |
| 진정인 | "진정인" |
| 대리인 | "대리인" + 본인 성명 병기 |

## 2. 법령 인용 형식

### 2.1 조문 인용 표준

> **표준**: `「○○법」 제○조 제○항 제○호`

- 법령명에 낫표(「」) 사용. 작은 따옴표·따옴표 금지
- 조·항·호는 한자식("제 조") 띄어쓰기 없이
- 별표 인용: `「○○법 시행규칙」 [별표 1]`

**예**:
- ⭕ 「행정절차법」 제22조 제3항에 따라
- ⭕ 「외국인근로자의 고용 등에 관한 법률」 제8조 제1항
- ❌ '행정절차법' 22조 3항 (낫표 X, 띄어쓰기 X)
- ❌ "행정절차법 제22조의3" (따옴표 X)

### 2.2 시행령·시행규칙

- 모법 → 시행령 → 시행규칙 순서로 인용 시:
  - `「○○법」 제○조 및 같은 법 시행령 제○조`
  - `「○○법 시행규칙」 [별표 ○] 제○호`

### 2.3 판례 인용

- `대법원 ○○○○. ○. ○. 선고 ○○○○○○호 판결` (날짜는 한자식 점)
- 헌법재판소: `헌법재판소 ○○○○. ○. ○. ○○헌가○ 결정`

### 2.4 행정청 행위·통지 인용

- `○○○○. ○. ○.자 ○○부장관의 ○○ 처분`

## 3. 수치·날짜 표기

- 수치: 천 단위 콤마 (`1,500,000원`)
- 비율: 백분율 + 표기 (`35%`, `95% 이상`)
- 기간: `2024. 3. 1.부터 2025. 2. 28.까지` (점 + 한 칸 띄움)
- 단일 날짜: `2024. 3. 1.` (말미 점 필수)
- 년·월·일 표기: `2024년 3월 1일`도 허용 (계획서 본문)
- 금액 표기 병기: `금 일천오백만원 (₩ 15,000,000)` — 계약·청구

## 4. 문장 구조

### 4.1 길이

- 본문 1문장 최대 100자 권장
- 종속절 3개 이상이면 분리
- 신청 취지·청구 취지는 한 문장 원칙 — 단, 가독성 우선

### 4.2 단락

- 단락당 3~5문장
- 단락 첫 문장은 주제문 (이하 무엇을 다루는지)
- 단락 마지막은 다음 단락으로의 연결 (계획서·진정서)

### 4.3 표·목록 사용

- 신청서·계획서: 항목 나열은 번호 매기기 (`1.`, `가.`, `(1)`)
- 청구 취지: 한 문장 원칙. 부득이 시 들여쓰기 목록
- 표는 식별 라벨 + 데이터로 명확히 (라벨 없는 표 금지)

## 5. 자리표시자 표준

- 형식: `[[필드명]]` 이중 대괄호 (`03-ui-model §2.1`의 EmptySlot 컨벤션과 일치)
- 한국어 라벨 — 영어 코드(`[[company_name]]`) 금지
- 빈 슬롯이 문장 중간이면 조사 처리: `[[회사명]]은(는)`, `[[금액]]을(를)`
- 단위가 명확하면 라벨에 포함: `[[고용 인원]]명`, `[[목표 매출]]억원`

## 6. 부처별 미세 차이 (참고)

> 행정사 검토 시 부처 표준에 따라 조정.

| 부처 | 특징적 표현 |
|---|---|
| 법무부 | "출입국관리법에 따라" 빈도 高 |
| 고용노동부 | "노사 합의에 따라" / "근로조건은…" |
| 환경부 | "환경영향 저감을 위하여" |
| 국토교통부 | "공익에 부합하는 범위에서" |
| 식약처 | "위해요소 관리를 위하여" |
| 행정안전부 | "주민 편익 증진" |

## 7. 진단 표현 (감점 키워드)

DraftWriter 출력이 다음을 포함하면 SelfReviewer 경고:

- "저희" / "우리" — 1인칭 위반
- "~인것 같습니다" — 추정 어조 (행정문서 부적합)
- "정말로" / "굉장히" — 강조 강세
- "~이거든요" / "~이에요" — 구어체 어미
- "그치만" / "근데" — 접속사 구어체
- "엄청난" / "최고의" — 마케팅 어조

## 8. 검토 요청

행정사 검토 시 우선순위:
1. **금지 표현 누락** — 위 7절에 더 추가할 표현
2. **부처별 차이 보강** — 6절의 부처 행 추가
3. **표준 어미 예외** — 어미 표준에서 빠진 도메인
4. **법령 인용 변형** — 헌법·국제조약 등 인용 형식
