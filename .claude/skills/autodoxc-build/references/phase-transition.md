# Phase 전환 절차 (B0 → B1 → B2 → B3 → B4)

각 Phase 종료 시 다음 Phase 진입 전 따르는 체크리스트 + lead-architect 산출물 기준.

> Phase 정의는 `docs/architecture/05-dev-team.md §5` 참조.

## 공통 종료 기준

모든 Phase 종료 전 확인:
- [ ] 해당 Phase의 작업 보드 항목 모두 `completed`
- [ ] 인터페이스 변경 ADR 모두 작성 완료
- [ ] qa-engineer 회귀 전부 그린
- [ ] 새 외부 의존성 결정사항 ADR 등록
- [ ] lead-architect 통합 보고서(`docs/architecture/reports/phase-{X}.md`) 작성
- [ ] 사용자 데모/승인

## 안티패턴

| 안 함 | 이유 |
|---|---|
| 종료 기준 미달 상태 Phase 전환 | 다음 Phase 작업 의존성 손상 |
| 회귀 그린 없이 진입 | 누적 회귀가 다음 Phase 진단 어렵게 |
| ADR 누락 + 진입 | 결정 이력 손실 |
| 외부 의존성 미결정 + 진입 | stub 누적 |
| 데모 없이 다음 Phase | 사용자 우선순위 동기 끊김 |
