## Should-NOT-Trigger (다른 도구가 적합한 케이스)

단순 질문·메모:
1. "autodoxc가 뭐 하는 프로젝트야?" — 단순 설명, 직접 응답
2. "RAG가 뭔지 일반론적으로 설명해줘" — 일반 개념, 직접 응답
3. "한국 행정문서 종류 일반적으로 어떤 게 있어?" — 정보 질문, 직접 응답

4. "내 친구가 행정사 시험 준비 중인데 공부법 알려줘" — 도메인 외, 직접 응답

6. "Claude API로 일반 챗봇 만들고 싶어" — claude-api 스킬

명백한 다른 작업:
7. "오늘 일정 알려줘" — 캘린더 도구
8. "내 깃허브 PR 리뷰 부탁" — review 스킬
9. "이 코드의 일반적인 버그 찾아줘" (autodoxc 컨텍스트 없이) — code-review 스킬

경계 모호 (판단 필요):

11. "한국법 사건 적용 법조 찾아줘" — `legal-basis-finder` 스킬 (autodoxc 빌드와 무관)

## 자가 점검 절차

1. 위 should-trigger 18개 중 각 케이스에 대해 `autodoxc-build` 스킬이 호출되는지 확인
2. should-NOT-trigger 11개 중 각 케이스에 대해 호출되지 **않는지** 확인
3. 경계 케이스(10, 11)는 대화 맥락에 따라 분기되는지 점검
4. 누락 또는 오트리거 발견 시 `SKILL.md`의 description 보강 — 키워드 추가 또는 부정 조건 추가
